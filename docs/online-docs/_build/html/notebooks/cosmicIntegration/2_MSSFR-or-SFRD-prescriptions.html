The MSSFR class¶

1. Introduction¶

The universe evolves over time, and as such the star formation rate and the metallicity of the star-forming gas change. The MSSFR class provides functionality to determine the amount of star formation that went into forming a system born at redshift \(z\), in a metallicity bin \(dZ\). In shorthand, we will use the term metallicity-specific star formation, either indicated with MSSFR or SFRD(Z,z), which is a function of metallicity \(Z\) and redshift \(z\). We will use MSSFR and SFRD(Z,z) interchangebly throughout this notebook: they are identical (just two different ways of naming the concept).

A schematic picture of how the SFRD(Z,z) is constructed is shown below (and reproduced in Figure 10 of the COMPAS methods paper):

Typically, the SFRD(Z,z) (MSSFR) is created by combining a star formation rate density (SFRD) with a metallicity distribution function \(\frac{dP}{dZ}\). The latter is constructed by combining a galaxy stellar-mass function (GSMF) with a mass-metallicity relationship.

For this script we assume that the SFR(z) and the metallicity distribution are independent of each other, hence the SFR is simply a normalization to the PDF of the metallicity distribution. Note that these distributions are averages over all galaxies at a specific redshift \(z\), hence do not apply to specific galaxies.

The script allows for several combinations of distributions. To this end we have to define the following steps:

1 - Create an instance of the MSSFR class where we define constants.

2 - Choose an SFR distribution.

3 - Choose a metallicity distribution.

Note that although this is meant to be used in the cosmic integration pipeline, this class can be independently used to plot distributions or as input for other calculations.

4 - Example of plotting.

5 - Examples of plotting the SFRD, the GSMF and the MZR relations (Figures 11, 13, and 14 from the COMPAS methods paper).

2. Creating an instance of the MSSFR class¶

To create an instance we need to define the following parameters:

verbose               = False

metallicityGrid       = None

binInLogSpace         = True
metallicityLowerLimit = 1E-90
metallicityUpperLimit = 1.0

logOH12sun            = 8.69
solarMetallicity      = 0.0142
cosmo                 = None

verbose

If True, prints progress messages during the calculation.

metallicityGrid

The grid assumed in order to create the metallicity bins \(\small{dZ}\). Automatically passed from the COMPAS class instance in a pipeline.

binInLogSpace

To create the bin edges you can either choose the midpoint between \(\small{Z_1}\) and \(\small{Z_2}\) or alternatively you can choose the midpoint between \(\small{\log{Z_1}}\) and \(\small{\log{Z_2}}\). Since most distributions work in the \(\small{\log{Z}}\) space it seemed more appropriate: the difference is not large. Otherwise a message indicating too few gridpoints in metallicity will be generated, since the curve changes a lot between \(\small{Z_1}\) and \(\small{Z_2}\).

metallicityLowerLimit

Since we integrate over \(\small{dZ}\) we need to define a lower limit for the metallicity mass fraction \(\small{Z}\) in our integral. We set the default close to zero, but non-zero to avoid division by zero errors.

metallicityUpperLimit

Since we integrate over \(\small{dZ}\) we need to define an upper limit for the metallicity mass fraction \(\small{Z}\) in our integral. We set the default to 1.0

logOH12sun

The assumed solar oxygen abundance (Asplund)

solarMetallicity

The assumed solar metallicity mass fraction (Asplund)

cosmo

The assumed universe (omega matter, omega lambda etc.) to translate between age/distance/time. If None, it is set to WMAP9. Automatically set by the CosmicIntegrator class instance in the cosmic integration pipeline.

4. Choose a metallicity distribution¶

There are two options for the metallicity distribution: MZ_GSMF and logNormal. These are described below.

4.1 MZ_GSMF: Mass metallicity and galaxy stellar mass function¶

The MZ_GSMF options combines a densitity distribution of galaxy stellar masses (GSMF) with a mass metallicity relation (MZ) in order to create a metallicity density function. In our code we translate from metallicity to mass to solve the integral, hence we use the inverse ZM functions. The original MZ functions are still in the source code for plotting/testing purposes

To use the MZ_GSMF option, set the metallicity prescription thus:

[6]:

MSSFR.Zprescription = 'MZ_GSMF'

Then choose the GSMF and MZ relation you require. To see the available options:

[7]:

MSSFR.printGSMFoptions()


For the `MZ_GSMF` option (i.e. self.Zprescription = 'MZ_GSMF'), the default GSMF prescription is:

    self.GSMFprescription = None

and current options are:

    'Panter et al. (2004) Single'
    'Furlong et al. (2015) Single'
    'Furlong et al. (2015) Double'

5. Discover and plot SFR fraction for a metallicity bin¶

The MSSFR class provides a function to return the fraction of SFR in a metallicity bin:

MSSFR.returnMSSFR(self, metallicity = None, agesBirth = None, redshiftBirth = None)

The code is constructed such that it calculates the MSSFR for all systems coming from a specific metallicity, given that you know their agesBirth or redshiftBirth (depending on the chosen MSSFR you need one or the other).

Note that if we define our own grid of metallicities and time at which we want to know the MSSFR, we can use this same function to visualise our assumed distribution. Of course we can also directly calculate any subfunctions to plot them :)

6.2 Plotting the galaxy stellar mass function GSMF¶

The blocks below show how to plot and use the galaxy stellar mass functions as shown in Figure 13 of the COMPAS methods paper.

From Neijssel et al. 2019:

Stellar mass density functions are given as Schechter functions of the form:

\[\phi(M) dM = \phi^{*} \left(\frac{M}{M_*}\right)^{\alpha} e^{\left(\frac{-M}{M_*}\right)} dM\]

\(\phi^*\) is the overall normalization

\(M_*\) determines turnover from power-law to exponential drop off

\(\alpha\) determines the slope power low for \(M\leq M_*\)

The prescription by Panter et al. 2004 is redshift independent, and therefore we just need to implement their notation. Alternatively we use a redshift dependent prescription by Furlong et al. They have both a single Schechter function in the same form as above, as well as a double Schechter function of the form:

\[\phi(M) dM = \left[ \phi_1 \left(t\right)^{\alpha_1} + \phi_2 \left(t\right)^{\alpha_2} \right] * e^{-\frac{M}{M_c}}\]

However, they only show the constants and several redshifts: hence most of this notebook is dedicated to turning this into a parametrized form to be used at any redshift.

6.2.1 Plotting the tabulated Furlong prescription¶

The plots in the paper such as Furlong et al. do NOT plot \(\small{M}\) vs \(\small\phi{dM}\); instead they plot \(\small\log{M}\) vs \(\small\phi{d}\log{M}\), so we need to transform the equation a bit

\[y = log_{10}(x)\]
\[\frac{dy}{dx} = \frac{1}{x}\]
\[dx = x dy\]

since \(x = \frac{M}{M_*}\) we have a \(\small{+1}\) in the alpha term for the Schechter \(\small{d\log_{10}}\) function.

def singleSchechterPerLog10(logM, logMc,  phi,  alpha):
    M  = 10**logM
    Mc = 10**logMc
    x  = M/Mc
    schechter = phi * ((x)**(alpha+1)) * np.exp(-x)
    return schechter
def doubleSchechterPerLog10(logM, logMc,  phi1,  a1, phi2, a2):
    M  = 10**logM
    Mc = 10**logMc
    x  = M/Mc
    schechter =(phi1 * ((x)**(a1+1))+  phi2 * ((x)**(a2+1))) * np.exp(-x)
    return schechter


def doubleSchechterPerLog10FurlongPolynomial(logM, z, power):
    logMc, phi1, a1, phi2, a2 = polynomialFitDoubleSchechter(z, power)
    schechter = ((phi1*(10**((a1+1)*(logM-logMc)))) + (phi2*(10**((a2+1)*(logM-logMc)))))\
               *(np.exp(-pow(10,logM-logMc)))
    return schechter


def doubleSchechterPerLog10FurlongLineair(logM, z):
    logMc, phi1, a1, phi2, a2, ratio = lineairFitDoubleSchechter(z)
    schechter = ((phi1*(10**((a1+1)*(logM-logMc)))) + (phi2*(10**((a2+1)*(logM-logMc)))))\
               *(np.exp(-pow(10,logM-logMc)))
    return schechter


def singleSchechterPerLog10FurlongPolynomial(logM, z, power):
    logMc, phi, alpha = polynomialFitSingleSchechter(z, power)
    schechter = phi*(10**((alpha+1)*(logM-logMc)))*(np.exp(-pow(10,logM-logMc)))
    return schechter


def singleSchechterPerLog10FurlongLineair(logM, z):
    logMc, phi, alpha = lineairFitSingleSchechter(z)
    schechter = phi*(10**((alpha+1)*(logM-logMc)))*(np.exp(-pow(10,logM-logMc)))
    return schechter

#In Furlong et al they note that a double schechter function
#fits the data better at the high galaxy stellar masses (log10M > 11)
def lineairFitSingleSchechter(z):

    try:
        nr = len(z)
    except:
        z  = np.array([z])
        nr = 1

    fitlogMc        = np.array([11.14, 11.11, 11.06, 10.91, 10.78, 10.60])
    fitphi1          = np.array([0.84, 0.84, 0.74, 0.45, 0.22, 0.12])*10**(-3)
    fita1            = np.array([-1.43, -1.45, -1.48, -1.57, -1.66, -1.74])


    fitredshifts = np.array([0.1, 0.5, 1.0, 2.0, 3.0, 4.0])

    fitvalues = [fitlogMc, fitphi1, fita1]
    thresholds = [0.0, 0.5, 1.0, 2.0, 3.0, 10000000]
    r          = []
    for nrv, values in enumerate(fitvalues):
        dz     = np.diff(fitredshifts)
        dydz   = np.divide(np.diff(values), dz)
        yvalues= np.zeros(nr)
        for nrz, redshift in enumerate(thresholds[:-1]):
            mask = (z>= thresholds[nrz]) & (z<=thresholds[nrz+1])
            if nrz == 0 :
                #interpolate from z0.5 down
                dz   = 0.5 - z[mask]
                interpolatedValue  = values[nrz+1] - np.multiply(dz, dydz[nrz])
            else:
                #interpolate up
                dz   = z[mask] - redshift
                interpolatedValue  = values[nrz] + np.multiply(dz, dydz[nrz])
            yvalues[mask] =  interpolatedValue
        r.append(yvalues)

    r[1][r[1] <0] = 0  #phi1
    return r[0], r[1], r[2]

def lineairFitDoubleSchechter(z):

    try:
        nr = len(z)
    except:
        z  = np.array([z])
        nr = 1


    fitredshifts = np.array([0.1, 0.5, 1.0, 2.0, 3.0, 4.0])
    fitlogMc     = np.array([10.95, 10.88, 10.74, 10.44, 10.19, 10.00])
    fitphi1      = np.array([1.45, 1.61, 1.51, 1.06, 0.63, 0.24])*10**-3
    fita1        = np.array([-1.31, -1.24, -0.98, -0.25, 0.23, 0.43])
    fitphi2      = np.array([0.0,  0.08,   0.48, 0.8, 0.61, 0.43])*10**-3
    fita2        = np.array([-2.22, -1.79, -1.62, -1.58, -1.64, -1.69])

    fitvalues  = [fitlogMc, fitphi1, fita1, fitphi2, fita2, ratioPhi1Phi2]
    thresholds = [0.0, 0.5, 1.0, 2.0, 3.0, 10000000]
    r          = []

    r[4][r[4]<-2] = -2 #a2
    return r[0], r[1], r[2], r[3], r[4], r[5]

def polynomialFitSingleSchechter(z, power):
    try:
        nr = len(z)
    except:
        z  = np.array([z])
        nr = 1

    fitredshifts = np.array([0.1, 0.5, 1.0,2.0, 3.0, 4.0])
    fitlogMc     = np.array([11.14, 11.11, 11.06, 10.91, 10.78, 10.60])
    fitphi       = np.array([0.84, 0.84, 0.74, 0.45, 0.22, 0.12])*10**(-3)
    fita         = np.array([-1.43, -1.45, -1.48, -1.57, -1.66, -1.74])

    fitvalues    = [fitlogMc, fitphi, fita]
    y            = []

    for nrv, values in enumerate(fitvalues):
        ysub         = np.zeros(nr)
        coefficients = np.polyfit(fitredshifts,values, power)[::-1]
        for nrpower, coefficient in enumerate(coefficients):
            ysub     = np.add(ysub, coefficient*np.power(z,nrpower))
        y.append(ysub)

    return y[0], y[1], y[2]

6.3 Plotting the MZR relations¶

The blocks below show how to use and plot the mass-metallicity relations as shown in Figure 14 of the COMPAS methods paper.

From Neijssel et al. 2019:

We get the relationship between the galaxy stellar mass and the oxygen to hydrogen number density. Using the relation between number density and mass fractino above we translate this to a metallicity. This metallicity is that of the interstellar gas. We assume that the stars that form at redshift \(\small{z}\) have the same metallicity as that of the interstellar gas.

Below we plot the different MZ relations from the MSSFR class as found in the respective papers. Given that(in the project) we want to use the inverse (ZM-relations), we test these too.
